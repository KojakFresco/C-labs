# Отчет по лабораторной работе: AVL-дерево

## Вариант

Реализовано упорядоченное дерево поиска для хранения элементов с уникальными строковыми ключами и вещественными значениями.

Особенности задачи:

- структура данных — `AVL-дерево`;
- ключ — непустая последовательность латинских букв длиной не более 6 символов;
- ключи упорядочиваются лексикографически;
- поддерживаемые операции: вставка, удаление, печать, поиск;
- предусмотрен ввод команд из текстового файла и вывод результата в текстовый файл;
- хотя бы один тестовый пример должен содержать дерево из 20 и более узлов.

## Решение

В проекте используется AVL-дерево, потому что оно сохраняет баланс после вставок и удалений, а значит гарантирует логарифмическую сложность основных операций.

### Основные части реализации

- `avl_tree.h` — объявление структур `AVLNode` и `AVLTree`, а также интерфейса операций.
- `avl_tree.c` — реализация вставки, удаления, балансировки, поиска и печати дерева.
- `main.c` — CLI-интерфейс.

### CLI-интерфейс

Поддерживаются два режима запуска:

1. **Интерактивный режим**
   - запуск без аргументов или с ключом `--interactive`;
   - команды:
     - `insert <key> <value>`
     - `delete <key>`
     - `find <key>`
     - `print`
     - `help`
     - `exit`

2. **Файловый режим**
   - запуск `--file <input.txt> <output.txt>`;
   - также сохранена обратная совместимость с форматом `<input.txt> <output.txt>`;
   - команды читаются в числовом формате:
     - `1 <key> <value>` — вставка;
     - `2 <key>` — удаление;
     - `3` — печать;
     - `4 <key>` — поиск.

### Принцип работы

- при вставке выполняется стандартная рекурсивная вставка с последующей балансировкой;
- при удалении узла дерево перестраивается и снова балансируется;
- поиск возвращает значение по ключу;
- печать выводит дерево в удобном для просмотра виде;
- в обоих режимах используется один и тот же обработчик операций, поэтому логика не дублируется.

## Тестирование

Для проверки реализован отдельный unit-test runner `test_avl.c`.

Покрытые сценарии:

1. вставка элемента и поиск существующего ключа;
2. поиск отсутствующего ключа;
3. удаление листа;
4. удаление узла с двумя потомками;
5. дерево из 20 узлов с проверкой баланса, высоты и вывода.

Тесты дополнительно проверяют:

- корректность BST-упорядочивания;
- корректность сохранения высоты узлов;
- AVL-баланс (`-1 .. 1`) для каждого узла;
- наличие результата поиска в текстовом выводе.

## Структура репозитория

Актуальная структура для этой лабораторной:

- `avl_tree.c` — реализация AVL-дерева;
- `avl_tree.h` — заголовок со структурами и прототипами;
- `main.c` — CLI и режим работы с файлами;
- `test_avl.c` — unit-тесты;
- `input.txt` — пример входных данных;
- `output.txt` — пример результата;
- `makefile` — сборка проекта и тестов;
- `report.md` — данный отчет.

## Запуск

### Сборка основной программы

```powershell
make build
```

### Интерактивный режим

```powershell
./main.exe --interactive
```

### Обработка команд из файла

```powershell
./main.exe --file input.txt output.txt
```

### Запуск unit-тестов

```powershell
make test
```
